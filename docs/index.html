The 3D OpenGL Renderer

An OpenGL Project developed for a univeristy course by Jaba Makharashvili.

Written Completely in Modern OpenGL

The complete rendering engine has been written in mordern OpenGL. Majority of OpenGL learning is always done in legacy OpenGL for simpilicity but I wanted to give myself a challenge and redid the complete project using purely modern OpenGL.

Learn more

Safety is Number One

OpenGL can be very hard to debug which makes it very dangerous as a lot of even simple problems result in just a blank screen. With this in mind I have every single OpenGL command wrapped in OpenGLCall() which is a macro that prints out the line of code that failed, the file it failed in, and the OpenGL error code. This way debugging is much easier and all problems can be quickly traced down to the exact line of code.

Learn more

Complexity Under The Surface

Even thought the engine itself once ran looks simple and only renders one backpack, the real complexity comes from the code and its architecture. Theres a fully functional logging system that only works in debug mode but no in release mode, every single openGL call is wrapped in safety, and there is a very smart use of buffers and shaders that legacy OpenGL does not even require to run the code. Due to this the code is very sophisticated, easily customizable and of course took a very long time to setup.
